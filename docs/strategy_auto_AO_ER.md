## 一、它一夜之间到底做了什么

整个流程的核心思想只有一句话： 你给一个股票代码，程序自动帮你完成策略发现、验证和压测。

具体来说，挂机一夜后它完成了这些事：

- 自动从上百种指标组合中，筛选出 Awesome Oscillator + Kaufman 效率比率的组合

- 生成一份完整的绩效报告

最终它给 AMT 找到的策略只有两个条件，下面我们逐一拆解。

## 二、两个核心指标：AO 与 ER

### 1. Awesome Oscillator（动量震荡指标）

AO 的计算非常简单：用「中位价」的短期均线减去长期均线。中位价是 (最高价 + 最低价) / 2 ，默认短周期 5、长周期 34。

```python
import pandas as pddef calculate_ao(df, short_period=5, long_period=34):    """    计算 Awesome Oscillator（AO）    df: 包含 'High' 和 'Low' 列的 DataFrame    short_period: 短期 SMA 周期，默认 5    long_period: 长期 SMA 周期，默认 34    """    df = df.copy()    # 中位价 = (最高价 + 最低价) / 2    df["MedianPrice"] = (df["High"] + df["Low"]) / 2    # 短期与长期简单移动平均    df["SMA_short"] = df["MedianPrice"].rolling(window=short_period).mean()    df["SMA_long"] = df["MedianPrice"].rolling(window=long_period).mean()    # AO = 短期均线 - 长期均线    df["AO"] = df["SMA_short"] - df["SMA_long"]    return dfdef ao_falling_consecutive(df, bars=3):    """    判断 AO 是否连续下降 bars 根 K 线    返回布尔序列，True 表示当根满足「连续下降」    """    df = calculate_ao(df)    # 当前 AO 是否比上一根更低    cond = df["AO"] < df["AO"].shift(1)    # 滚动窗口内全部为 True，说明连续下降    return cond.rolling(bars).sum() == bars
```

入场逻辑 ：AO 连续 3 根 K 线下降时买入。直觉上这是「在短期动量走弱时介入，押注后续延续」。

> 小知识： rolling(bars).sum() == bars 是判断「连续 N 次都满足某条件」的常用技巧。因为布尔值在求和时 True 记为 1，只有当窗口内全部为 True 时，和才等于窗口长度。

### 2. Kaufman 效率比率（趋势效率）

ER 衡量的是「价格走得有多直」。它等于「一段时间的净位移」除以「同段时间内每日波动绝对值之和」，取值范围在 0 到 1 之间。值越接近 1，说明趋势越干净、越有效率；越接近 0，说明在反复震荡。

```python
def kaufman_efficiency_ratio(df, period=14):    """    计算 Kaufman 效率比率（ER）    period: 回看周期，默认 14    """    df = df.copy()    # 净变化：period 周期内收盘价的绝对位移    change = df["Close"].diff(period).abs()    # 总波动：每日变化绝对值在 period 周期内的总和    volatility = df["Close"].diff().abs().rolling(period).sum()    # 效率比率 = 净变化 / 总波动    er = change / volatility    return erdef er_trend_confirmed_up(df, period=14, level=0.7):    """    判断趋势是否「高效且向上」    条件：ER 高于阈值 level，且收盘价较上一日上涨    """    df = df.copy()    df["ER"] = kaufman_efficiency_ratio(df, period)    return (df["ER"] > level) & (df["Close"] > df["Close"].shift(1))
```

出场逻辑 ：当 ER 高于 0.7 且价格上涨时卖出，也就是「在市场以高效率单边运行时离场」。

## 三、用 vectorbt 完成回测

有了入场和出场信号，就可以交给 vectorbt 这个高性能回测库。需要特别注意的是：信号要 向后平移一根 （ shift(1) ），用「次日开盘价」成交，这样才能避免「用当天还没发生的信息做决策」这种典型的未来函数错误。

```python
import vectorbt as vbt# 生成入场 / 出场信号df["entry_signal"] = ao_falling_consecutive(df)df["exit_signal"] = er_trend_confirmed_up(df)# 关键点：信号向后平移一根，用次日开盘价成交，避免未来函数shift_entries = df["entry_signal"].shift(1).astype(bool).fillna(False)shift_exits = df["exit_signal"].shift(1).astype(bool).fillna(False)# 构建组合：用开盘价成交，考虑手续费与滑点pf = vbt.Portfolio.from_signals(    close=df["Open"],      # 以开盘价成交    entries=shift_entries,    exits=shift_exits,    init_cash=100_000,     # 初始资金 10 万    fees=0.001,            # 手续费 0.1%    slippage=0.002,        # 滑点 0.2%    freq="1d")# 打印绩效统计print(pf.stats())
```

> 易错点提醒：手续费（fees）和滑点（slippage）一定要加上。很多新手回测出「神仙曲线」，往往就是忽略了交易成本。一笔交易看似只差 0.3%，但高频累积下来足以吞掉全部利润。

## 四、为什么必须用「向前优化」

这是全文最重要的一节，也是新手最容易踩的坑。

普通回测往往是骗人的。 你在一段历史数据上调参，又把这套参数拿回同一段数据上验证，自然能得到一条漂亮的资金曲线。但这不是策略，这叫 过拟合（曲线拟合） ——参数只是被「背」出来了，换一段数据立刻失效。

向前优化（Walk-Forward Optimization） 的做法完全不同：

1. 用 4 年数据训练，找出最优参数
2. 在 接下来 1 年 从没见过的数据上测试，记录真实的样本外结果
3. 把窗口向前滑动，重复上述过程，直到走完全部历史

这样得到的每一年都是真正的「样本外」结果，程序从未用明天的数据去做今天的决策。这才是检验一个策略是否真有效的关键。

## 五、它给 AMT 找到的结果

挂机一夜后，向前优化给出的样本外成绩如下：

- 总收益（向前优化）：4,792.7%
- 基准（买入持有）：2,561.4%
- 夏普比率：0.710
- 索提诺比率：1.139
- 胜率：72.2%
- 盈亏比：2.562
- 最大回撤：-87.01%
- 总交易次数：24 年共 55 笔
- 平均每笔预期收益：90,756 美元

策略 24 年的总收益接近买入持有的两倍，胜率超过 70%，而且交易极少，平均每年只有约 2 笔。

但这里必须冷静地看一个数字： -87% 的最大回撤是真实的。 AMT 是一只 REIT（房地产信托），在 2022 年加息周期里被重创。也就是说，即便长期收益亮眼，中途也可能经历账户腰斩再腰斩。任何在这只标的上的策略都必须正视这一点。

## 六、蒙特卡洛压力测试：区分「稳健」与「运气」

向前优化之后，程序又自动做了一件事：把 55 笔交易的 先后顺序随机打乱 1000 次 ，看看结果在不同排列下还能不能站得住。

5th / 50th / 95th 百分位的结果如下：

- 总收益：39.80% / 4,583.19% / 164,234.42%
- 最大回撤：-90.42% / -72.49% / -54.01%
- 夏普比率：0.23 / 0.59 / 0.93

中位情形仍有 4,583% 收益、0.59 夏普；即便最悲观的 5th 百分位也还是正收益 39.8%。

这正是 稳健策略与幸运策略的分水岭 ：如果一个策略只在某一种特定的交易顺序下才赚钱，它就是脆弱的；如果它能在 1000 次随机打乱后依然成立，说明这个优势是结构性的，而不是侥幸。

> 延伸理解：蒙特卡洛重排背后的假设是「单笔交易之间相对独立」。它能帮你估计回撤和收益的分布区间，但它无法消除策略本身依赖的市场环境风险——比如上面那次 -90% 的极端回撤，依然可能在现实中发生。

## 七、必须清醒认识的边界

最后要强调，这是一份 由程序自动生成的回测结果，不是真实的实盘记录 。

- 历史业绩不代表未来收益
- -87% 那种级别的回撤场景是真实存在的，未来仍可能重演
- 它展示的是「自动化策略发现 + 严格压力测试在技术上可行」，而不是「建议你照搬去交易某只股票」

程序负责搜索，你负责审阅和判断——这才是正确的人机分工。

## 总结

回到 Python 学习者的视角，这个案例真正值得带走的，不是某个具体的收益数字，而是一整套 严谨的量化方法论 ：

1. 指标实现 ：AO 是均线之差，ER 用「净位移 / 总波动」衡量趋势效率，都可以用 pandas 几行写出
2. 避免未来函数 ：信号 shift(1) ，用次日价格成交，回测才有意义
3. 计入成本 ：手续费与滑点必须纳入，否则曲线是假的
4. 向前优化 ：用样本外数据检验，才能识破过拟合
5. 蒙特卡洛压测 ：打乱顺序看分布，区分稳健与运气
6. 敬畏风险 ：再亮眼的收益，也要先看回撤能不能扛得住

把这套流程内化，比记住任何「神奇指标」都更有价值。把重复劳动交给代码，把判断留给自己，这正是用 Python 做量化的意义所在。
